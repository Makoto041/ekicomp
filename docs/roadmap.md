# EKICOMP ロードマップ

## Phase 1：MVP（最小構成）— 目安 4〜6週間

**ゴール：「駅にチェックインして地図で見える」を動かす**

- プロジェクトセットアップ（Next.js + Ktor + Supabase）
- 駅マスターデータ投入（国土数値情報 → DB）
- Google ログイン
- 手動チェックイン（駅名検索 → チェックイン）
- 地図表示（全駅ピン、訪問済み/未訪問の色分け）
- 駅詳細（訪問日時の表示）
- 全体達成率（シンプルな数値表示）
- PWA基本対応

## Phase 2：記録を充実させる — 目安 3〜4週間

**ゴール：「メモ・写真・スポットで思い出を残せる」**

- メモ・写真投稿
- スポット記録（居酒屋等）
- 路線別進捗 + プログレスバー
- Apple ログイン追加
- 地図タップチェックイン

## Phase 3：ゲーミフィケーション — 目安 3〜4週間

**ゴール：「バッジとランダム提案で継続モチベーションを作る」**

- バッジシステム（定義 + 条件判定 + 通知）
- ランダム駅提案（フィルタ付き）
- 統計ダッシュボード（月別グラフ、ストリーク）
- GPSチェックイン

## Phase 4：ソーシャル機能 — 目安 4〜6週間

**ゴール：「友達と競い合える、共有できる」**

- フレンド機能
- ランキング
- SNSシェア（画像生成）
- GPS自動通知

## Phase 5：パートナー機能 & 一般公開 — 将来

**ゴール：「彼女との共有マップ、一般公開」**

- ペアリング・共有マップ
- ペアバッジ
- パフォーマンス最適化・不正対策
- ネイティブアプリ化検討
